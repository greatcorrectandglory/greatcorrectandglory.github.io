# 官网内容策略

## 页面定位

首页服务于客户的 30 秒理解：这是面向电子制造和研发验证的通用自动化测试平台，不是开发文档、源码说明或内部交付手册。首屏直接回答三个问题：

1. 场景：电子制造、研发验证、产线测试软件。
2. 能力：产品配置包、插件化测试步骤、多工位执行、硬件资源绑定、结果追溯、MES/云平台对接。
3. 价值：减少重复开发、缩短新产品导入时间、降低现场交付复杂度。

## 文案逻辑

页面从客户问题进入，而不是从技术栈进入：

- Hero 首屏：一句话定位平台和核心价值。
- 客户痛点：描述重复开发、设备接入不统一、追溯困难和系统集成复杂。
- 平台解决方案：把痛点映射到配置包、插件、资源绑定、结果追溯。
- 核心架构图：用公开抽象表达“配置 -> 运行时 -> 数据与集成”，不暴露内部源码结构。
- 三类用户入口：测试员、硬件工程师、软件工程师分别看到自己的使用价值。
- 典型应用场景：覆盖研发验证、产线测试、新产品导入、现场交付和质量追溯。
- 核心功能清单：集中承载搜索关键词和能力完整性。
- 部署与集成方式：说明静态官网不依赖后端，平台本体面向本地工站和企业系统。
- FAQ：处理开源边界、适用团队、与脚本的差异、静态部署问题。
- 联系/演示申请：引导通过 GitHub 联系，同时提醒不要公开提交敏感信息。

## SEO 关键词策略

首页关键词覆盖以下查询意图：

- 类目词：通用测试平台、自动化测试平台、产线测试软件、电子产品测试。
- 技术词：Python PyQt 测试平台、插件化测试系统、多工位测试。
- 集成和数据词：MES 对接、测试数据追溯、云平台对接。
- 对标词：OpenHTF、OpenTAP。

关键词放置原则：

- H1 只保留一个，承载核心定位。
- `title` 同时包含平台名称、场景和测试软件属性。
- `meta description` 控制在 120 到 160 字，覆盖场景、能力和价值。
- 关键词自然分布在首屏、解决方案、功能清单和 FAQ 中，避免堆砌。
- Open Graph 使用同一套标题和描述，便于社交平台分享。

## 信息安全边界

公开页面不写入以下内容：

- private 仓库源码、内部目录和实现细节。
- 客户名称、真实产品型号、工厂地址、网关地址、协议字段或公司内部系统信息。
- 产品配置包里的参数、设备 VID/PID、真实接口路径、账号或密钥。

如需展示案例，应使用匿名行业场景，例如“多工位产线测试”“研发样机验证”“新产品导入”，不要使用可反推客户或产品的信息。
